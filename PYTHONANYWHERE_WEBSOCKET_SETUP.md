# إعداد WebSocket على PythonAnywhere - دليل شامل

## ⚠️ المشكلة الحالية:
```
Not Found: /ws/chat/order/1/
```

هذا يعني أن WebSocket URL غير مُعرّف في PythonAnywhere Dashboard.

## ✅ الحل الكامل:

### الخطوة 1: التحقق من الحساب

**مهم جداً:** WebSocket متوفر فقط للحسابات المدفوعة على PythonAnywhere.

- ✅ **حساب مدفوع:** WebSocket يعمل
- ❌ **حساب مجاني:** WebSocket قد لا يعمل بشكل كامل

### الخطوة 2: إعداد WebSocket في Dashboard

1. **سجل الدخول** إلى PythonAnywhere Dashboard
2. اذهب إلى **Web** tab (في الأعلى)
3. ابحث عن قسم **WebSocket** (في الأسفل، تحت Static files)
4. إذا لم ترَ قسم WebSocket:
   - تأكد من أن لديك حساب مدفوع
   - أو راسل الدعم الفني لـ PythonAnywhere

5. **أضف WebSocket URLs التالية:**

#### WebSocket 1 - للشات:
```
URL pattern: /ws/chat/order/<int:order_id>/
Handler: mr_delivery.asgi.application
```

#### WebSocket 2 - لتحديثات الطلبات:
```
URL pattern: /ws/orders/shop/<int:shop_owner_id>/
Handler: mr_delivery.asgi.application
```

6. **احفظ** التغييرات
7. اضغط على **Reload** (أعلى الصفحة)

### الخطوة 3: التحقق من المسار

تأكد من أن:
- **Handler** يشير إلى: `mr_delivery.asgi.application`
- إذا كان المشروع في مجلد فرعي، استخدم المسار الكامل
- مثال: إذا كان المشروع في `/home/username/mr_delivery/`، استخدم `mr_delivery.asgi.application`

### الخطوة 4: التحقق من ASGI File

تأكد من أن `mr_delivery/asgi.py` موجود ويحتوي على:

```python
import os
from django.core.asgi import get_asgi_application
from channels.routing import ProtocolTypeRouter, URLRouter
from channels.middleware import BaseMiddlewareStack
import shop.routing
from shop.middleware import JWTAuthMiddleware

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'mr_delivery.settings')

django_asgi_app = get_asgi_application()

application = ProtocolTypeRouter({
    "http": django_asgi_app,
    "websocket": BaseMiddlewareStack(
        JWTAuthMiddleware(
            URLRouter(
                shop.routing.websocket_urlpatterns
            )
        )
    ),
})
```

### الخطوة 5: تثبيت المكتبات

في **Bash Console** على PythonAnywhere:

```bash
# تأكد من أنك في مجلد المشروع
cd ~/mr_delivery  # أو المسار الصحيح

# تثبيت المكتبات المطلوبة
pip3.10 install --user channels channels-redis daphne redis

# أو إذا كنت تستخدم Python 3.11
pip3.11 install --user channels channels-redis daphne redis
```

### الخطوة 6: التحقق من Settings

في `mr_delivery/settings.py`، تأكد من:

```python
# 1. ASGI_APPLICATION موجود
ASGI_APPLICATION = 'mr_delivery.asgi.application'

# 2. CHANNEL_LAYERS مُعرّف
CHANNEL_LAYERS = {
    'default': {
        'BACKEND': 'channels_redis.core.RedisChannelLayer',
        'CONFIG': {
            "hosts": [('redis.pythonanywhere.com', 6379)],
        },
    },
}

# 3. channels في INSTALLED_APPS
INSTALLED_APPS = [
    'daphne',  # يجب أن يكون أولاً
    'channels',
    # ... باقي التطبيقات
]
```

### الخطوة 7: إعادة تحميل التطبيق

1. في **Web** tab
2. اضغط على **Reload** (أعلى الصفحة)
3. انتظر بضع ثوان حتى يتم إعادة التحميل

### الخطوة 8: اختبار الاتصال

بعد إعادة التحميل، جرّب الاتصال من `frontend_test.html`:

```javascript
const ws = new WebSocket(
    'wss://mrdelivery.pythonanywhere.com/ws/chat/order/1/?token=your_token'
);
```

## 🔍 استكشاف الأخطاء:

### خطأ: "Not Found: /ws/chat/order/1/"

**الأسباب المحتملة:**
1. WebSocket URLs غير مُعرّفة في Dashboard
2. Handler غير صحيح
3. الحساب مجاني ولا يدعم WebSocket

**الحل:**
- أضف WebSocket URLs في Dashboard
- تأكد من صحة Handler
- تأكد من أن لديك حساب مدفوع

### خطأ: "Module not found: channels"

**الحل:**
```bash
pip3.10 install --user channels channels-redis daphne redis
```

### خطأ: "Redis connection failed"

**الحل:**
- تأكد من إعداد `CHANNEL_LAYERS` في `settings.py`
- تأكد من أن Redis متوفر (للمستخدمين المدفوعين فقط)

### خطأ: "Handler not found"

**الحل:**
- تأكد من أن `mr_delivery.asgi.application` موجود
- تحقق من المسار في Dashboard
- تأكد من أن `asgi.py` يحتوي على `application` variable

## 📸 لقطات شاشة توضيحية:

### في PythonAnywhere Dashboard:

1. **Web Tab** → **WebSocket Section** (في الأسفل)
2. أضف:
   ```
   URL: /ws/chat/order/<int:order_id>/
   Handler: mr_delivery.asgi.application
   ```
3. اضغط **Save**
4. اضغط **Reload**

## ✅ قائمة التحقق:

- [ ] حساب مدفوع على PythonAnywhere
- [ ] WebSocket URLs مُعرّفة في Dashboard
- [ ] Handler صحيح: `mr_delivery.asgi.application`
- [ ] المكتبات مثبتة: `channels`, `channels-redis`, `daphne`, `redis`
- [ ] `ASGI_APPLICATION` موجود في `settings.py`
- [ ] `CHANNEL_LAYERS` مُعرّف في `settings.py`
- [ ] `asgi.py` يحتوي على `application` variable
- [ ] تم إعادة تحميل التطبيق (Reload)

## 🆘 إذا استمرت المشكلة:

1. **تحقق من Error Log:**
   - في **Web** tab → **Error log**
   - ابحث عن أخطاء متعلقة بـ WebSocket أو channels

2. **تحقق من Server Log:**
   - في **Web** tab → **Server log**
   - ابحث عن أخطاء في بدء التشغيل

3. **راسل الدعم:**
   - إذا كان لديك حساب مدفوع ولا يعمل WebSocket
   - راسل دعم PythonAnywhere مع تفاصيل المشكلة

4. **بديل مؤقت:**
   - استخدم REST API للشات بدلاً من WebSocket
   - أو استخدم Polling للتحقق من الرسائل الجديدة

## 📝 ملاحظات مهمة:

1. **HTTPS/WSS:** على PythonAnywhere، استخدم `wss://` وليس `ws://`
2. **Token:** تأكد من أن Access Token صحيح ولم ينتهِ صلاحيته
3. **Order ID:** تأكد من وجود الطلب في قاعدة البيانات
4. **Permissions:** تأكد من أن الطلب يخص صاحب المحل المتصل
